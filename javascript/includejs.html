Including External JS file in HTML

Javascript code can be written internally with HTML page but it is not a Good Practice.

Always write Javscript in a seperate file and include it in HTML page.

Prefer External Javascript files because :

  1. Reuse - External Code can be used in multiple pages
  2. Caching - Browser Caches External JS files, making pages to load quickly and saves user data.
  3. Readibility - Seperate code is readable and easy to debug.
  4. Debugger - Its easy to run browser debugger in JS file. You cannot run debugger in HTML page

Note: It is advisable to not use multiple seperate files because it increases number of HTTP/HTTPS request and increases load time. Developers should try to keep Network request as low as possible.

How to include Javascript file

You can include External JS files either inside head tag or before body closing tag. However it does affects the page load time.

Including Javascript
<head>
   <script src="js/myfile.js"></script>
</head>

To avoid Render-Blocking in your webpage, always try to include javascript file before closing of body tag. Advantages are :

  1. Page do not wait for the javascript to be downloaded and executed.
  2. HTML is parsed quickly and loading appears to be quick.
  3. You need HTML and CSS before Javscript. So incase your js file is large , it wont block Page page content from rendering.
Javscript before closing body tag
<body>
   ...
   ...
   <script src="js/myfile.js"> </script>
</body>

Usind Async and Defer tag

Without async or defer, browser will run your script immediately, before rendering the elements that's below your script t

  1. Without async or defer, browser will run your script immediately, before rendering the elements that's below your script tag.
  2. With async (asynchronous), browser will continue to load the HTML page and render it while the browser load and execute the script at the same time.
  3. With defer, browser will run your script when the page finished parsing. (not necessary finishing downloading all image files)
Async and Defer
   <script src="js/myfile.js" async> </script>
OR
   <script src="js/myfile.js" defer> </script>

What if Javascript is disabled ?

Use noscript tag for handling disabled javascript scenario, Theoratacially this tag also works for browser which do not support javascript.

Frankly speaking, i am yet to see a modern browser which do have support for js.

noscript usage
   <noscript> Javascript is disabled in your Browser </noscript>

Using Charset

Words and sentences in text are created from characters. Examples of characters include the Latin letter á or the Chinese ideograph 請 or the Devanagari character ह.

You may not be able to see some of the characters in this page because you don't have the necessary fonts. If you click on the place where you expected to see a character you will link to a graphic version. This page is encoded in UTF-8. Characters that are needed for a specific purpose are grouped into a character set (also called a repertoire). (To refer to characters in an unambiguous way, each character is associated with a number, called a code point.)

Using Charset
   <script src="js/myfile.js" charset="external file encoding type"> </script>
UTF-8 and ISO-8859-1 are widely used encoding types.
