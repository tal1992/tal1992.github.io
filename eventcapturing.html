Events Handling in Javascript

Each browser event handler is registered in a context. When you call addEventListener as shown previously, you are calling it as a method on the whole window because in the browser the global scope is equivalent to the window object. Every DOM element has its own addEventListener method, which allows you to listen specifically on that element.

Adding event Listener - addEventListener
<button>I am the element with id button <button>
<script>
  var button = document.querySelector("#button");
  button.addEventListener("click", function() {
    console.log("Div clicked.");
  });
</script>

The example attaches a handler to the #button node. Thus, clicks on the element cause that handler to run, whereas clicks on the rest of the document do not.

One time event Listener - removeEventListener
<button>Just-once button<button>
<script>
  var button = document.querySelector("button");
  function once() {
    console.log("Done.");
    button.removeEventListener("click", once);
  }
  button.addEventListener("click", once);
</script>

Event Bubbling vs Capturing

Event capturing

When you use event capturing

               | |
---------------| |-----------------
| element1     | |                |
|   -----------| |-----------     |
|   |element2  \\ /          |     |
|   -------------------------     |
|        Event CAPTURING          |
-----------------------------------
the event handler of element1 fires first, the event handler of element2 fires last.

Event bubbling

When you use event bubbling

               / \\
---------------| |-----------------
| element1     | |                |
|   -----------| |-----------     |
|   |element2  | |          |     |
|   -------------------------     |
|        Event BUBBLING           |
-----------------------------------
the event handler of element2 fires first, the event handler of element1 fires last.
addEventListener(type, listener, useCapture)
Event bubbling is enabled by default. To use event capturing set "useCapture" parameter to true.

Useful Methods to control events

  • event.preventDefault: Cancels the event if it is cancelable, without stopping further propagation of the event.
  • event.stopPropagation: Prevents further propagation of the current event.
  • event.stopImmediatePropagation: Prevents other listeners of the same event from being called.
